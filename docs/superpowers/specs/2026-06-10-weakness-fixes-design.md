# Design: Privacy & Integrity Hardening

**Date:** 2026-06-10
**Status:** Approved by user (scope and design confirmed in session)

## Context

A code assessment identified four sharp, low-risk weaknesses. This effort fixes all four. Explicitly **out of scope** (tracked as separate future efforts):

- Full encryption at rest — SQLCipher would violate the zero-external-dependency constraint. The documented stance is: FileVault (system-level) + POSIX 0700 directory permissions + optional Time Machine exclusion. This stance is recorded in an ADR.
- Semantic search scaling (linear scan over all embeddings).
- Decomposition of `ContentView.swift` / `TimelineManager.swift`.
- Broad test-suite expansion beyond the tests written for these fixes.

## 1. ClipboardCapture: concealed/transient pasteboard filtering

**File:** `MementoCapture/Sources/ClipboardCapture.swift`

Password managers (1Password, Bitwarden, KeePassXC, Apple Passwords) mark copied secrets with de facto standard pasteboard marker types (nspasteboard.org). Today these land in plaintext in the database when clipboard monitoring is enabled.

- Change signature to `getNewClipboardContent(from pasteboard: NSPasteboard = .general) -> String?`. The parameter exists for testability (tests use a uniquely named `NSPasteboard`); the production call site in `CaptureService` is unchanged.
- After updating `lastChangeCount` (so the same item is not re-evaluated), return `nil` if `pasteboard.types` contains any of:
  - `org.nspasteboard.ConcealedType`
  - `org.nspasteboard.TransientType`
  - `org.nspasteboard.AutoGeneratedType`
- Marker types are a private static constant `[NSPasteboard.PasteboardType]`.
- Always on. No setting — this is a correctness/privacy fix, not a preference.

## 2. StorageProtection (new enum namespace)

**New file:** `MementoCapture/Sources/StorageProtection.swift`

Follows the existing stateless-enum-namespace convention (`StorageCleaner`, `StorageMigrator`).

- `static func applyDirectoryPermissions(to url: URL)` — sets POSIX permissions `0o700` on the storage directory via `FileManager.setAttributes`. Logs `AppLog.warning` on failure (e.g. exFAT volumes); never throws.
- `static func setExcludedFromBackup(_ excluded: Bool, on url: URL)` — sets `URLResourceValues.isExcludedFromBackup`. Logs warning on failure; never throws.

**New setting:** `excludeFromBackup: Bool`, UserDefaults key `"excludeFromBackup"`, **default `true`** (excluded).

- `Settings`: new `@Published var excludeFromBackup` following the existing didSet-persist pattern; didSet also applies the new value immediately to the current `storageURL` — sets the flag when `true`, clears it when `false`.
- `SettingsWindow`: toggle in the storage section.
- `L` (Capture `Localization.swift`): new label + description strings, Swedish/English.
- `docs/SETTINGS.md`: new row documenting the setting.

**Call sites:**

- `CaptureService.prepareCaptureResourcesIfNeeded()`: after the storage directory is ensured, call `applyDirectoryPermissions` and `setExcludedFromBackup(Settings.shared.excludeFromBackup)`.
- `CaptureService.switchStoragePath(to:)`: after successful migration, apply both to the new path. The old (now empty) directory may retain its backup-exclusion flag; this is harmless and accepted.

Works identically on both distribution channels (direct `~/.cache/memento` and App Store group container).

## 3. Foreign key enforcement

SQLite ships with `foreign_keys` OFF, so the declared `FOREIGN KEY` constraints are currently decorative. The pragma is **per connection**.

- `Database.openDatabase()`: execute `PRAGMA foreign_keys=ON` after open.
- `StorageCleaner.cleanup()`: execute the pragma after its own `sqlite3_open`. Its delete order is already child-first (`CONTENT_FTS` → `EMBEDDING` → `CONTENT` → `FRAME`) and remains valid.
- `LegacyTimelineMigration`: inspect during implementation; if it opens a writing connection, add the pragma there too.
- `INSERT OR REPLACE INTO FRAME`: frame ids are assigned monotonically from `MAX(id)`, so id conflicts do not occur in normal operation. Under FK enforcement, a conflicting REPLACE that would orphan child rows now fails loudly (logged) instead of silently pairing old OCR blocks with a new frame. This is the accepted, intended behavior change.

The Timeline module reads only; FK enforcement on read connections is irrelevant and Timeline is not modified.

## 4. Quiet migrations

`Database.createTables()` currently runs `ALTER TABLE ADD COLUMN` blindly; on every launch after the first, each statement fails and logs a SQL error.

- New private helper `columnExists(table:column:) -> Bool` using `PRAGMA table_info(<table>)`.
- `PRAGMA` statements cannot take bound parameters; the table name is interpolated **only from private string constants within `Database`**, never from external input. This documented exception to the no-interpolation rule is noted in CLAUDE.md.
- Each migration `ALTER TABLE` is guarded by `!columnExists(...)`.
- Net effect: zero SQL error log lines on routine launches; behavior otherwise identical.

## 5. Tests (TDD, existing `MementoCaptureTests` target)

Written test-first. All use temp paths/named pasteboards; no global state pollution.

- **ClipboardCaptureTests** (`@MainActor`): concealed-marked content → `nil`; plain text → captured; unchanged `changeCount` → `nil`; `isEnabled == false` → `nil`.
- **DatabaseMigrationTests**: create a legacy-schema database (e.g. `FRAME` without `url` column) at a temp path, then init `Database` on it → column is added; init `Database` twice on a fresh temp path → idempotent, all expected columns present.
- **StorageCleanerTests**: temp DB seeded with frames/content/embeddings + dummy `.mp4` files; run `cleanup` with cutoff under FK enforcement → expected rows and whole-video files deleted, partial videos retained.
- **StorageProtectionTests**: temp directory → permissions read back as `0o700`; backup-exclusion flag reads back `true` after set, `false` after clear.

## Performance

No hot-path impact above noise: the pasteboard type check and FK lookups on insert are each well under 1 ms. No change to the Performance Budget table beyond a note on the SQLite insert row if measured ≥1 ms.
