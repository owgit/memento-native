Privacy by default

Your archive stays on your Mac.

Memento is built for predictable capture: clear controls, local storage, and pauses for sensitive or low-value contexts.

Local storage

Data is stored locally by default at ~/.cache/memento. No cloud backend is required for capture or search.

Pause anytime

Switch between Recording and Paused directly from the menu bar Control Center.

Private browsing guard

Private and incognito windows can trigger best-effort automatic pause when detected.

No telemetry pipeline

The app does not require analytics or a server to make your visual history searchable.

Permissions explained

macOS asks before any screen or browser context is indexed.

The app remains explicit about what each permission does and what quality changes when it is denied.

Screen Recording

Required so Memento can read visible screen content. Without it, capture and OCR search cannot do their core job.

Automation for browser context

Used to read active tab URL and title from supported browsers. Without it, capture still works, but search results may miss URL/title context.

Open FAQ
Why does Memento need Screen Recording?

macOS requires this permission for any app that reads visible screen content.

Why does browser access need Automation?

Automation lets Memento add active tab URL and title context for richer results. Without it, that metadata may be missing.

Where is data stored?

The default direct-build location is ~/.cache/memento. Storage and retention are configurable.

Why can updates ask for an admin password?

The updater replaces the app in /Applications, which often requires elevated privileges on macOS.
